## üìù Opening an Issue

### üêõ Bug Reports

Found a bug? Help me fix it by providing detailed information:

**What to include:**
- **Clear title** - Describe the issue briefly (e.g., "Editor crashes when pasting large text")
- **Steps to reproduce** - Exact steps to make the bug happen
- **Expected behavior** - What should happen
- **Actual behavior** - What actually happens
- **Environment details**:
  - Browser (Chrome, Firefox, Safari, etc.) and version
  - Operating System (Windows, macOS, Linux)
  - JustMD version (if applicable)
- **Screenshots/GIFs** - Visual evidence helps a lot!
- **Console errors** - Check browser console (F12) for error messages

**Example:**
```markdown
**Bug:** Document title doesn't save after editing

**Steps to reproduce:**
1. Open a document
2. Click on the title and edit it
3. Press Enter or click away
4. Refresh the page

**Expected:** Title should be saved
**Actual:** Title reverts to previous value

**Environment:** Chrome 120.0, macOS 14.1
**Screenshot:** [attached]
```

### ‚ú® Feature Requests

Have an idea to improve JustMD? I'd love to hear it!

**What to include:**
- **Clear description** - What feature would you like to see?
- **Use case** - Why is this feature important? How would you use it?
- **Examples** - Reference other apps that do this well (if applicable)
- **Mockups/sketches** - Visual representation helps (optional but appreciated!)

**Example:**
```markdown
**Feature:** Add markdown syntax highlighting in the editor

**Use case:** As a user writing complex markdown documents, I would like to see
syntax highlighting for headers, bold, italic, and code blocks in the editor pane
to make it easier to spot formatting mistakes.

**Example:** VS Code's markdown preview does this well by highlighting different
syntax elements with subtle colors.
```

## ‚úÖ Issue Guidelines

To help me help you faster:

### DO ‚úÖ
- **Be respectful and patient** - Remember, this is a free, open-source project
- **Provide details** - More information = faster resolution
- **Use proper formatting** - Code blocks, headers, lists make issues easier to read
- **One issue per report** - Don't combine multiple bugs or requests
- **Update your issue** - If you find more info or a workaround, share it!
- **Close resolved issues** - If your problem is solved, close the issue

### DON'T ‚ùå
- **Don't spam** - Bumping issues every day doesn't help
- **Don't be rude** - Constructive criticism is welcome, but be kind
- **Don't post duplicates** - Search first, then post
- **Don't share sensitive data** - Never post passwords, API keys, or personal information
- **Don't expect instant support** - I'm one person, not a 24/7 support team
